# codemap

Статический анализатор Python-пакетов: **ест исходники → строит queryable граф кода** —
фактическую карту, которую потребляют генерация документации, AI/CLI-навигация (RAG) и
аудит/чистка. Философия: один канонический граф-стор → много рендеров.

**Статус:** 🟢 M0+M1 реализованы (API-surface + queryable граф, 13/13 тестов). См. **[DESIGN.md](DESIGN.md)** и **[BACKLOG.md](BACKLOG.md)**.

## Зачем изолирован

`codemap` — dev-инструмент **общего назначения** (работает над любым пакетом), не часть
библиотеки `bquant`. Живёт отдельным каталогом в корне репо, **не попадает в wheel** и развязан
с внутренностями пакета (статический `ast`-разбор, без импорта рантайма). Прицел — вынос в
**отдельный репо-продукт**, когда созреет. Сейчас `bquant` — подопытный вход (`codemap build ../bquant`).
