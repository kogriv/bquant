## MACD-зоны: суть правки дефолта (G4)

**Проблема:** методология (`macd_research.md`) определяет зону по знаку **линии MACD**
(`MACD>0` бык / `<0` медведь) — длинные стабильные зоны. Дефолт кода детектировал по
**гистограмме** (`macd_hist` = MACD−signal) — короткие волатильные зоны. Все прошлые
case-study считались на гистограмме.

**Эвиденс:** 1h 72→31 зон (медиана длит. 12→20), m15 87→33 (8→25).

**Решение:**
- `bquant/analysis/zones/presets.py::analyze_macd_zones` — новый параметр
  `zone_basis` (`'line'` по умолчанию → `indicator_col='macd'`; `'histogram'` →
  `'macd_hist'` для legacy/быстрого варианта). **Дефолт теперь линия.**
- Docstring-примеры в `pipeline.py` поправлены (`macd_histogram` → `macd`; заодно
  `macd_histogram` — несуществующая колонка).
- Примеры `04/08` выровнены на линию.
- Депрекейтнутый `MACDZoneAnalyzer` — оставлен на гистограмме (backward-compat, G11).

**Влияние:** прошлые артефакты (case-study 05/06) считались на гистограмме; новые
рисёрчи (S1+) — на линии. При сравнении с legacy это учитывать.
