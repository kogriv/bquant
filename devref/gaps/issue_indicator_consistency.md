# Issue: Indicator implementation inconsistency (MACD / RSI)

**Дата:** 2026-07-06
**Тип:** gap / correctness
**Приоритет:** 🔴 High — блокирует «шлюз корректности» из `devref/architecture/revival_plan_2026-07.md` (этап 1)
**Статус:** 🟡 Open — замеры сделаны, решение по эталону за владельцем

---

## 1. Проблема

BQuant содержит несколько реализаций одних и тех же индикаторов
(`custom/`, `library.py`, `preloaded/`, `library/talib.py`). Трёхисточниковая
модель (внешние / внутренние / preloaded) — осознанный дизайн и остаётся. Но
**собственная реализация даёт числа, расходящиеся с TA-стандартом**, потому что
использует `ewm()` с дефолтным `adjust=True` вместо рекурсивного EMA
(`adjust=False`). Весь смысл проекта — статистика по зонам этих индикаторов,
поэтому расхождение не косметическое.

Первопричина (root cause):
- `custom/macd.py:81-82` — `close.ewm(span=...).mean()` (adjust=True).
- `custom/rsi.py:73-74` — `gains.ewm(alpha=1/period).mean()` (adjust=True), что
  **не является** сглаживанием Уайлдера (Wilder RMA требует `adjust=False`).
- Те же паттерны продублированы в `library.py` и `calculators.py`.

## 2. Замеры (факты)

Данные: встроенный сэмпл `tv_xauusd_1h`, 1000 баров. Эталоны:
- **MACD reference** = рекурсивный EMA `ewm(span, adjust=False)`;
- **RSI reference** = Wilder RMA `ewm(alpha=1/period, adjust=False)`;
- **preloaded** = числа, зашитые TradingView в сэмпл.
- `talib` в окружении отсутствует (опциональна) — сравнение против pandas-ta и
  ручного эталона.

Скрипт замера: воспроизводим (см. раздел 6).

### 2.1. MACD (12, 26, 9)

| Сравнение | mean\|Δ\| (линия) | max\|Δ\| | corr |
|-----------|------------------:|---------:|-----:|
| **pandas_ta** vs reference(adjust=False) | 0.039 | 2.60 | 0.99947 |
| custom vs reference(adjust=False) | 0.200 | 6.32 | 0.99209 |
| custom vs pandas_ta | 0.146 | 7.52 | 0.99387 |
| pandas_ta vs preloaded(TradingView) | 0.044 | 2.93 | 0.99932 |
| custom vs preloaded(TradingView) | 0.193 | 5.84 | 0.99303 |

**Вывод:** `pandas_ta` практически совпадает с рекурсивным эталоном и с
TradingView. `custom` отклоняется в ~4–5× сильнее.

**Влияние на детекцию зон** (знак `macd_hist` = bull/bear классификация,
стратегия `zero_crossing`):

| Реализация | Число зон (сегментов по знаку) |
|-----------|-------------------------------:|
| custom | **83** |
| reference (adjust=False) | 79 |
| pandas_ta | **73** |

- Баров с иным знаком гистограммы: custom vs reference — **9/1000 (0.90%)**,
  custom vs pandas_ta — 4/1000 (0.40%).
- **Ключевой факт:** число зон различается на **73 → 83 (~14%)**. Для проекта,
  считающего статистику *по зонам*, это материально: та же гипотеза на тех же
  данных даст разные N, длительности и распределения в зависимости от того, какой
  реализацией посчитан индикатор.

### 2.2. RSI (14)

| Сравнение | mean\|Δ\| | max\|Δ\| (пункты RSI) | corr |
|-----------|----------:|---------------------:|-----:|
| **pandas_ta** vs Wilder reference | **0.000** | **0.000** | **1.000000** |
| custom vs Wilder reference | 0.444 | **54.38** | 0.97505 |
| custom vs pandas_ta | 0.444 | 54.38 | 0.97505 |

- `pandas_ta.rsi` **точно** = Wilder (расхождение 0.0).
- `custom` RSI отклоняется до **54 пунктов RSI** (шкала 0–100) — это не
  «другая конвенция», а фактическая ошибка сглаживания.
- **Влияние на пороговую детекцию** (RSI 70/30, стратегия `threshold`):
  **12/999 баров (1.20%)** попадают в иной режim overbought/neutral/oversold →
  другие зоны.

### 2.3. Побочная находка

- `preloaded` (TradingView) сам по себе не совпадает идеально ни с pandas_ta
  (mean|Δ| RSI 0.66), ни с эталоном — иной вендор + округление зашитых чисел.
  Это **ожидаемо и допустимо**: preloaded — это «чужие» числа, их нельзя сделать
  идентичными расчётным. Цель consistency — только для *расчётных* источников
  (custom ↔ library ↔ talib), а не для preloaded.
- Непоследовательность имён колонок: custom RSI → `rsi_14`, custom MACD →
  `macd`/`macd_signal`/`macd_hist`. Стоит унифицировать контракт выходных колонок.

## 3. Решение по эталону (развилка — за владельцем)

`pandas_ta` уже в обязательных зависимостях и совпадает с TA-стандартом
(RSI точно, MACD ~совпадает). Два варианта:

- **Вариант A (рекомендуемый): `custom/*` делегирует в pandas-ta под капотом.**
  Минус велосипед, один источник правды для расчёта, числа = индустрия.
  Custom-классы остаются как тонкие обёртки (сохраняют трёхисточниковый контракт),
  но внутри зовут pandas-ta. Риск: жёсткая привязка расчёта к pandas-ta (он и так
  обязателен).
- **Вариант B: оставить собственный расчёт, но исправить на `adjust=False`.**
  Сохраняет независимость от библиотеки; custom MACD станет = reference, RSI =
  Wilder. Плюс тест эквивалентности против pandas-ta как страховка. Минус —
  дублирование логики остаётся (просто корректное).

В обоих вариантах результат в числах одинаковый (совпадёт с TA-стандартом).
Разница — в том, держим ли мы собственную формулу или делегируем.
