## 1. Рамка и что изменилось (пересмотр revival-плана)

`revival_plan_2026-07.md` откладывал уборку, сплит и англ-версию — потому что тогда целью
была **петля рисёрча** внутри одного репо. С тех пор произошёл **сплит**: аналитика и поиск
альфы переехали в приватную лабу `bquearch`. Поэтому:

- **`bquant` теперь = публичный тулкит.** Его задача — быть чистым, «сухим», пригодным для OSS
  и для ИИ-CLI пакетом. Рисёрч/методология живут в лабе.
- Ограничения revival-плана под эту новую цель **пересмотрены** (уборка из «отложить» →
  в «делать»). Anti-todo частично снят (см. §4).
- Зависимость по-прежнему строго `bquearch → bquant`. Пакет ничего не знает про лабу.

## 2. North-star (куда идём)

**Пакет как чистое ядро + документация как отдельный продукт с несколькими интерфейсами.**

Целевые интерфейсы доки (не сейчас, но как ориентир): **ReadTheDocs · свой сайт ·
AI-friendly дока** (напр. `llms.txt`/чанкуемый markdown/машиночитаемый API-reference).

Ключевой принцип, делающий это дёшевым: **один источник истины → много рендер-таргетов.**
RTD, сайт и AI-render — это три сборки над одним чистым, оттестированным источником, а не три
разных дока. Отсюда — **исходник доков остаётся в пакете** (рядом с кодом, для парити), а
продуктом становится **витрина/сайт**, не исходник.

**Дальний прицел — RAG-на-минималках для ИИ-CLI:**
- **Дока = семантический слой:** что делает, зачем, как применять.
- **Код-граф = структурный слой:** что от чего зависит, что откуда течёт, кого заденет правка.
- Вместе они закрывают то, на чём ИИ обычно галлюцинирует (дока не знает структуру, греп не
  знает смысла). «Минималки» = чистый markdown + граф в JSON, навигируемые CLI-агентом; вектор-
  стор/эмбеддинги — только если упрёмся.

**Следствие для приоритетов:** **контент раньше интерфейсов.** Сушка — это не гигиена ради
гигиены, а **фундамент под AI-слой** (грязная дока → бесполезный RAG). Это же снимает старый
страх «рефакторинг усыпляет»: у уборки появился внешний результат ниже по течению.

## 3. Арка (как всё связано)

```
сушка пакета + тестируемая дока (substrate)
        ↓
   чистый единый источник
        ↓
{RTD · сайт · AI-friendly render}  +  машиночитаемый код-граф
        ↓
   RAG-на-минималках для ИИ-CLI
```

## 4. Принятые решения / принципы (locked)

| Тема | Решение |
|---|---|
| Целевой язык | **Английский как единственный**, инкрементально (тронул файл — перевёл). Синхронную пару RU↔EN **не заводим** (двойная поддержка = ловушка). |
| Исходник доков | **В пакете** (doc-driven/парити). Наружу выносим сайт/рендеры, не исходник. |
| Косметика GitHub | `.gitattributes` `linguist-documentation` для `docs/**` + gitignore билдов — вместо сплита ради статистики. |
| Код-граф | Не первым шагом. Когда дойдём — тестируем твои внешние тулы/сервисы, а мои детерминированные методы (grimp/AST) = **эталон-бенчмарк** для их проверки. Артефакт графа — машиночитаемый (под RAG). |
| Метод работы | Мелкие шаги, полное понимание каждого, PR-на-батч, ничего необратимого без согласия. |

**Anti-todo (НЕ делаем):** формальный сплит на 3 репо; синхронная пара RU↔EN; переписывание
кэша/логов/nb/performance на внешние либы; вынос исходника доков; новые process-completion-report'ы.
