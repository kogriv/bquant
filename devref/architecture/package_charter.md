# BQuant — Хартия пакета (конвенция)

**Тип:** Позиция / рабочая конвенция (durable — меняется редко, осознанно).
**Введена:** 2026-07-24 (заменяет стратегическую ось `revival_plan_2026-07.md`).
**Живой план под эту хартию:** `package_roadmap_2026-07.md`.

Этот документ фиксирует **что такое пакет, чем он не является и как мы с ним работаем.**
Он не про конкретные задачи (это роадмап) — он про рамку, в которой задачи выбираются.

---

## 1. Что такое пакет

**BQuant — публичный самостоятельный инструментарий количественного анализа рынков:**
честный, воспроизводимый zone-analysis для *любого* индикатора (universal pipeline
`analyze_zones()`), плюс данные/индикаторы/визуализация/статистика вокруг.

Ценность — **методологический каркас**, а не «денежный принтер»: пайплайн, которому
можно верить, включая честные *отрицательные* результаты.

## 2. Чем пакет НЕ является

- **Не продукт для поиска альфы / прогнозирования.** Поиск альфы, forecasting-слой,
  трансляция рисёрчей — это домен приватной лабы `bquearch`, не пакета.
- **Не витрина рисёрчей.** Демо в пакете показывают *функциональность*, а не альфу.

## 3. Отношения с лабой `bquearch`

- **Пакет живёт своей жизнью.** Его развитие определяется его собственными целями
  (см. §4), а не потребностями лабы.
- **Лабу обслуживаем ТОЛЬКО по явному запросу от неё.** Нет запроса — нет работы «на
  лабу». Не тащим в пакет альфа-логику; не тащим проактивно контент в лабу.
- **Одностороннее правило (сохраняется):** инстанс, работающий с пакетом, **не правит
  конфиг лабы** (pyproject, AGENTS.md, PROGRAM.md и т.п.). Обмен — через размещение
  контента по договорённости, и только когда это запрошено.

## 4. North-star (куда пакет растёт сам по себе)

Приоритет — **пакет как аккуратный, понятный, самодокументированный фундамент:**

1. **Документация как продукт** — единый источник правды → много рендеров (сайт/RTD,
   AI-friendly формат). Фундамент заложен уборкой (доки согласованы с кодом,
   doc-parity guardrail стоит).
2. **Граф кода + RAG-на-минималах** — карта связей и поиск, чтобы люди и ИИ-агенты
   быстро ориентировались.
3. **Качество / DX** — зелёный сьют, чистая навигация, «тесты одной командой».

**Отложено осознанно** (anti-todo, переносится из старого плана):
- Английский как primary — до появления внешней аудитории (сейчас mixed ru/en — ок).
- Разбивка репо — монорепо правильно для соло без внешних потребителей.
- v3.0.0 (выпил legacy `MACDZoneAnalyzer`) — веха, когда созреет; не гнать.

## 5. Рабочие конвенции (как ведём работу)

- **Трейслоги.** Работа документируется в `changelogs/CHANGE_TRACE_LOG_YYYY-MM-DD.md`
  (плотный вид, `COMMIT DIVIDER`); суммаризация в `CHANGELOG.md` при паблише.
- **Doc-parity guardrail.** `tests/unit/test_docs_parity.py` — авторитетная проверка
  «доки = код». Переименовал API/файл → чини доку в том же изменении.
- **Инструкции агентам.** `AGENTS.md` — единый источник; `CLAUDE/GEMINI/VIBE.md` —
  тонкие указатели через `@AGENTS.md`.
- **Релиз.** Версия в 2 местах (`pyproject.toml` + `bquant/__init__.py`); сборка/twine
  через `venv_bquant`; `origin` пушит на GitHub+GitLab; PyPI-токен в `~/.pypirc`,
  никогда не коммитить. Подмодуль `bquest` **никогда не стейджим**.
- **Верификация перед действием.** Удаляешь — сперва проверь (трекинг, ссылки,
  импорты); публикуешь наружу — сперва подтверди зелёный сьют и билд.
