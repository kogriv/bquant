## 2. Определение зоны (РЕШЕНИЕ)

MACD-термины: **линия MACD** = EMA_fast − EMA_slow; **сигнальная** = EMA(линии);
**гистограмма** = линия − сигнальная.

- **Методология** (`macd_research.md`): зона по знаку **линии MACD** (`MACD>0` бык /
  `MACD<0` медведь), границы — пересечение нуля. Длинные, стабильные зоны.
- **Текущий дефолт кода** (пресеты, враппер, все case-study): `indicator_col='macd_hist'`
  — зона по знаку **гистограммы** (= пересечение MACD с сигнальной). Короткие,
  волатильные зоны. **Противоречит методологии.**

**Решение:** исследование ведём на зонах по **линии MACD** — `detect_zones('zero_crossing',
indicator_col='macd')`.

Эвиденс (min_duration=2, zigzag+global):

| Датасет | по `macd_hist` | по `macd` (выбрано) |
|---|---|---|
| XAUUSD 1h | 72 зоны, медиана 12, max 39 | **31 зона, медиана 20, max 98** |
| XAUUSD m15 | 87 зон, медиана 8, max 60 | **33 зоны, медиана 25, max 85** |

**Бонус для темы свингов:** длинные зоны = больше свингов на зону = богаче внутренняя
структура. Зон меньше (n≈16 bull/датасет — тем важнее дисциплина малых n), но каждая
зона — более информативный объект для анализа позиции/последовательности свингов.

## 3. Ландшафт гипотез (таксономия)

Гипотезы порождаются **системно** из пересечения осей, а не подбираются ad-hoc.
H1–H5 из `macd_research.md` — частные клетки этой таблицы.

- **Признак свинга:** амплитуда · длительность · скорость · направление (rally/drop) ·
  порядковый номер · число свингов в зоне.
- **Обусловливание (condition):**
  - тип зоны (bull / bear);
  - **позиция в зоне** (относительное время 0→1; относительный ценовой путь);
  - **класс формы зоны** (кластер shape: острый импульс / плавный тренд / затухающая волна);
  - режим «продолжение зоны vs её конец».
- **Горизонт:** внутри свинга · внутри зоны · форвард после свинга · форвард после зоны.

Каждая заполненная клетка = кандидат-гипотеза. Ландшафт — это карта, по которой мы
движемся, а не список «любимых» гипотез.

## 4. Дисциплина исследования (evidence-based, для малых n)

По канону evidence-based TA (Аронсон). На малых n p-value — описательная величина,
не приговор. Правила:

1. **Exploration ≠ confirmation.** Разведочные прогоны генерируют гипотезы (без
   заявлений о значимости); подтверждение — отдельно, с фиксированной гипотезой.
2. **Effect size + bootstrap CI** вместо p-хакинга.
3. **Репликация между 1h и m15** как главный фильтр: оставляем то, что воспроизводится
   на обоих датасетах.
4. **Всё против null-модели** (перемешанные свинги/зоны, Monte-Carlo — есть в
   `ValidationSuite`).
5. **Поправка на множественность** при многих гипотезах (Holm-Bonferroni — есть в
   `HypothesisTestSuite`).

Выход этапа — не «правило торговли», а: характеристики зон MACD · список зон-кандидатов
в гипотезы · **список дыр в тулките**.

## 5. Структура программы: два слоя A → B

Ось — не «лестница по одной оси за раз», а **два связанных слоя**: **A — описание
(фундамент)**, **B — прогнозируемость (цель)**. Единица анализа — **вся зона как
траектория свингов** (не отдельная нога/откат). Только **направление зоны** (в bull —
long-only снимаемые импульсы, в bear — short-only).

### Слой A — Фундамент: структура и классы завершённых зон (дескриптивная статистика)

Нужен, потому что прогнозировать класс/структуру нельзя, не зная, какие они бывают.

- **A1 — Классификация.** Кластеризуются ли завершённые зоны в несколько устойчивых
  архетипов по структуре (num_legs, harvest, профиль «размер ноги × позиция»,
  shape skew/kurtosis, `rally_to_drop_ratio`)? Сколько классов, устойчивы ли 1h↔m15.
- **A2 — Различимость.** Отличаются ли архетипы по торговому смыслу (harvest, макс. нога
  и её позиция, глубина откатов, ratio) — стоит ли их различать.
- **A3 — Масштаб с длиной.** num_legs / harvest / макс. нога как функция длины зоны —
  дают ли длинные зоны пропорционально больше снимаемого хода и точек набора.
- **A4 — Профиль по классам.** Находка S1 (импульс затухает, откат растёт к концу) —
  общая или живёт лишь в части классов.

Дескрипторы завершённой зоны (это **статистики структуры**, НЕ симулированный P&L):
- **harvest** = сумма импульсных ног в направлении зоны (сколько хода в свою сторону
  вообще есть в структуре);
- **max adverse** = макс. просадка в любой точке зоны (риск удержания).
- ← экскурсия (бывш. MFE/MAE) переехала сюда как дескриптор **всей зоны**, single-
  pullback версия отброшена (теряла траекторию).

### Слой B — Цель: прогнозируемость из частичной зоны (статистика зависимости, НЕ бэктест)

Прогноз = **приор по истории зон + подмешивание частичных признаков на баре k**.
Формулируется как **статистика зависимости**, не как обученный предиктор и не как
симуляция стратегии.

- **Приор:** `P(класс)`, распределения итоговых величин (harvest, num_legs, длительность)
  по всей истории.
- **Условное:** `P(итог | частичные признаки на баре k, зона дожила до k)`.
- **Информация ранней части:** расхождение условного и приора (MI / дистанция) —
  сколько дают первые k баров.
- **Кривая информации:** рост этого расхождения с k (и с долей f). Это и есть
  «динамически/асимптотически улучшается», но как **статистика по истории**.

Мишени: итоговый **класс** (категор.) + **структура** (остаточный harvest / число ног /
длительность) + **дожитие** (сколько зона ещё продлится).

Правильные срезы:
- **k абсолютное** (реалтайм, счёт баров) и **доля f** зоны (структурно-сопоставимо) — оба;
- **Survivorship:** на баре k выборка = только зоны, дожившие до k → с ростом k выборка
  сжимается к длинным. Это корректное условие («живая зона на баре k уже дожила»), но
  честно репортим `n(k)` и смещение;
- **Стратификация:** тип зоны, итоговый класс, ТФ (репликация), корзина длины.

Оговорки (чтобы не самообмануться):
- **Утечка:** класс определяется по полной зоне, но признаки на k — строго из первых k
  баров (без ATR/свингов из будущего);
- **Малые n:** мерим *зависимость* (MI, условные распределения, корреляции); не дробим
  на много ячеек и не обучаем предиктор;
- **Это НЕ бэктест:** никаких equity-кривых, жадного набора, накопленного P&L — только
  статистика фактической истории в разных срезах.

## 6. Ключевой тул под это (research-driven допил, G10)

**k-нарезаемый экстрактор структуры зоны** — одна функция, дающая дескрипторы зоны как
для **полной** зоны (Слой A), так и для **префикса первых k баров** (Слой B), без утечки
будущего. Текущие `SwingMetrics` — только агрегаты на полную зону; нужен префиксный
режим. Плюс **форвардная разметка дожития/конца зоны** для мишеней B. Оба —
переиспользуемые пайплайн-шаги/скиллы, рождённые задачей.
