## 1. Рамка и цель (важно для методологии)

На данном этапе цель — **НЕ альфа**, а: (1) созревание методологии, (2) созревание
тулкита (пайплайны/скиллы, допил под реальные нужды), (3) **характеризация одиночного
индикатора** («что даёт голый MACD») через структуру его зон.

Данные — **осознанно** два малых встроенных датасета (`tv_xauusd_1h`,
`mt_xauusd_m15`, по 1000 баров). Это стенд для отладки методологии, а не база для
статистически робастной альфы. Прямое следствие для дисциплины (раздел 4):
статистика здесь **эксплоративная, не подтверждающая**.

**Наив уже пройден.** Свёрнутая метрика «доходность зоны от начала до конца» отвергнута

работаем сразу со *структурой* внутри зоны, а не с одной цифрой. Отдельной
«описательной» стадии без учёта позиции нет — она не несёт ценности.

## 2. Определение зоны (РЕШЕНИЕ)

MACD-термины: **линия MACD** = EMA_fast − EMA_slow; **сигнальная** = EMA(линии);
**гистограмма** = линия − сигнальная.

- **Методология** (`macd_research.md`): зона по знаку **линии MACD** (`MACD>0` бык /
  `MACD<0` медведь), границы — пересечение нуля. Длинные, стабильные зоны.
- **Текущий дефолт кода** (пресеты, враппер, все case-study): `indicator_col='macd_hist'`
  — зона по знаку **гистограммы** (= пересечение MACD с сигнальной). Короткие,
  волатильные зоны. **Противоречит методологии.**

**Решение:** исследование ведём на зонах по **линии MACD** — `detect_zones('zero_crossing',
indicator_col='macd')`.

Эвиденс (min_duration=2, zigzag+global):

| Датасет | по `macd_hist` | по `macd` (выбрано) |
|---|---|---|
| XAUUSD 1h | 72 зоны, медиана 12, max 39 | **31 зона, медиана 20, max 98** |
| XAUUSD m15 | 87 зон, медиана 8, max 60 | **33 зоны, медиана 25, max 85** |

**Бонус для темы свингов:** длинные зоны = больше свингов на зону = богаче внутренняя
структура. Зон меньше (n≈16 bull/датасет — тем важнее дисциплина малых n), но каждая
зона — более информативный объект для анализа позиции/последовательности свингов.

## 3. Ландшафт гипотез (таксономия)

Гипотезы порождаются **системно** из пересечения осей, а не подбираются ad-hoc.
H1–H5 из `macd_research.md` — частные клетки этой таблицы.

- **Признак свинга:** амплитуда · длительность · скорость · направление (rally/drop) ·
  порядковый номер · число свингов в зоне.
- **Обусловливание (condition):**
  - тип зоны (bull / bear);
  - **позиция в зоне** (относительное время 0→1; относительный ценовой путь);
  - **класс формы зоны** (кластер shape: острый импульс / плавный тренд / затухающая волна);
  - режим «продолжение зоны vs её конец».
- **Горизонт:** внутри свинга · внутри зоны · форвард после свинга · форвард после зоны.

Каждая заполненная клетка = кандидат-гипотеза. Ландшафт — это карта, по которой мы
движемся, а не список «любимых» гипотез.

---

## 4. Дисциплина исследования (evidence-based, для малых n)

По канону evidence-based TA (Аронсон). На малых n p-value — описательная величина,
не приговор. Правила:

1. **Exploration ≠ confirmation.** Разведочные прогоны генерируют гипотезы (без
   заявлений о значимости); подтверждение — отдельно, с фиксированной гипотезой.
2. **Effect size + bootstrap CI** вместо p-хакинга.
3. **Репликация между 1h и m15** как главный фильтр: оставляем то, что воспроизводится
   на обоих датасетах.
4. **Всё против null-модели** (перемешанные свинги/зоны, Monte-Carlo — есть в
   `ValidationSuite`).
5. **Поправка на множественность** при многих гипотезах (Holm-Bonferroni — есть в
   `HypothesisTestSuite`).

Выход этапа — не «правило торговли», а: характеристики зон MACD · список зон-кандидатов
в гипотезы · **список дыр в тулките**.

## 5. Лестница сложности (стартуем сразу со структуры)

Сложность растёт **добавлением одной оси обусловливания за раз**. Каждый уровень —
законченный, самодостаточный, публикуемый кусок. К правилам не прыгаем до S3.

- **S1 — Структура (позиция + последовательность).** ✅ выполнено.
  Разложить зоны на отдельные свинги; для каждого — позиция в зоне (rel_time 0→1,
  rel_price), направление, амплитуда, порядковый номер. Распределения, **обусловленные
  позицией и типом зоны**. Вопрос: *«какие свинги, где и когда внутри зоны»*.
  *(Находка: зоны линии MACD сильно лагают — 38% bull-зон закрываются в плюс; эдж в
  экскурсии/тайминге, а не в hold. См. `docs/analytics/zones/s1_swing_anatomy_case_study.md`.)*

- **S1b — Экскурсионная состоятельность (MFE/MAE от отката).** ← следующий шаг.
  **Переопределение метрики состоятельности:** не симметрия амплитуд ног (слепа к
  таймингу), а **MFE/MAE от точки входа на откате** — вход = дно отката (trough для
  bull / вершина для bear), исход = ход в свою сторону (MFE) vs просадка/разворот
  (MAE), в % и ATR-нормированные. Вопрос: *«какой reward/risk даёт вход на откате и
  как он зависит от позиции отката в зоне»*. Это outcome-переменная (y) для S3.

- **S2 — Классы зон.**
  Обусловить свинг-структуру на **класс формы зоны** (кластеризация shape-метрик).
  Вопрос: *«в каких классах зон какая структура свингов»*. (Ось «Форма + Свинги» из
  философии.)

- **S3 — Различение «откат vs конец зоны».**
  Форвардная разметка: для каждого контр-свинга — зона после него продолжилась или
  закончилась? Отличает коррекцию-точку-входа от начала разворота (ловушки). Вопрос:
  *«этот контр-свинг — вход или ловушка»*. **Здесь описание переходит в альфу.**

- **S4 — Действие.**
  Только тут появляются не-наивные правила-кандидаты (например: «контр-свинг в bull-зоне
  класса X в первых 60% зоны, амплитуда < медианы») — и только как гипотезы на валидацию.

## 6. Дыры в тулките, которые вскроет исследование (research-driven допил)

- **S1:** per-swing **позиция внутри зоны**. Текущие `SwingMetrics` — агрегаты на зону
  (counts, avg amplitudes, ratios); по-штучной координаты свинга нет. → новый
  аналитический тул/пайплайн.
- **S1b:** **MFE/MAE от точки входа на откате** (trough/peak), в % и ATR-нормированные,
  окно = до конца зоны или до противоположного пивота. Отсутствует.
- **S3:** **форвардная разметка исхода** зоны/свинга (контр-свинг → продолжение/разворот),
  связанная с MFE/MAE (низкий MFE + высокий MAE = «ловушка/конец зоны»). Отсутствует.
- Все — кандидаты в переиспользуемые пайплайн-шаги / «скиллы», рождённые реальной
  задачей, а не спекулятивно.

## 7. Первый шаг — S1 «Анатомия свингов зон MACD»

- **Данные:** оба датасета; зоны по линии MACD (`indicator_col='macd'`), `zigzag`+`global`.
- **Построить** per-swing датасет: `zone_id`, `zone_type`, `swing_order`, `direction`,
  `amplitude_pct`, `duration_bars`, `rel_time` (позиция начала свинга в зоне 0→1),
  `rel_price` (позиция по ценовому пути зоны).
- **Анализ:** распределения rally/drop отдельно для bull/bear; распределения,
  обусловленные `rel_time` (ранняя/средняя/поздняя треть зоны); репликация 1h↔m15;
  сравнение с null-моделью (перемешанные позиции).
- **Выход:** отчёт (Quarto/markdown) + backlog допила тулкита под S1 (per-swing позиция).
