# Метод-стек и тул-стек (максимальный, пошаговый)

**Дата:** 2026-07-06
**Назначение:** единый зафиксированный компас — какие методы и инструменты (библиотеки)
мы используем на каждом шаге программы `swing_structure_research_program.md` (слои A→B).
Это **меню**, из которого мы берём по шагам, а не «поставить всё сразу».

**Фазы данных:**
- **Фаза метода (сейчас, малые данные):** отлаживаем механику методов/тулов на текущих
  1000-барных сэмплах. **Статвыводы не делаем** (n≈30 зон/датасет).
- **Фаза данных (следующая, большие данные):** 100k+ баров, несколько инструментов/ТФ —
  только здесь классы/прогноз/кривые информации становятся статистически состоятельными,
  и включаются walk-forward / OOS / кросс-актив.

**Инсталляция инкрементальная:** ставим пакет к тому шагу, где он впервые нужен (колонка
«когда»). Уже есть: `numpy`, `pandas`, `scipy`, `scikit-learn`, `statsmodels`.

## 1. Слой A — Классификация / структура завершённых зон

| Шаг | Метод | Что даёт | Тул | Когда | Малые данные? |
|---|---|---|---|---|---|
| A-i | KMeans + silhouette на агрегатах | baseline-классы | scikit-learn | ✅ сделано | механика — да |
| A-ii | **DTW + TimeSeriesKMeans / k-medoids** на посл-ти ног и пути | классы с уважением к порядку/длине | **tslearn** | **сейчас** | механика — да |
| A-ii+ | soft-DTW barycenters | прототипы кластеров | tslearn | сейчас | да |
| A-iii | **Feature-bank**: catch22 / tsfresh на пути → кластер | принципиальные признаки вместо ручных | **tsfresh** / `pycatch22` (или sktime `Catch22`) | сейчас/A-late | да |
| A-iii+ | **SAX → BOSS / WEASEL** (символизация, bag-of-patterns) | дискретные паттерны формы | **pyts** | сейчас/A-late | да |
| A-iv | **HMM / Markov-switching / GMM** над ногами | классы как сигнатуры режимов (импульс/коррекция/истощение) | hmmlearn; statsmodels `MarkovRegression`; sklearn `GaussianMixture` | A-late / общий с B | частично |

## 2. Слой B — Прогноз структуры из частичной зоны (цель)

Формулировка — **статистика зависимости** (приор + частичное на баре k), не бэктест.

| Шаг | Метод | Что даёт | Тул | Когда | Малые данные? |
|---|---|---|---|---|---|
| B-i | Условные распределения + **mutual information** префикса vs итог, по k/доле f | кривая информации (baseline) | sklearn `mutual_info_*`; scipy | фаза данных (механика — сейчас) | механика — да |
| B-ii | **HMM forward-filtering** | `P(класс|префикс)` + ожидаемая остаточная структура + дожитие | hmmlearn | фаза B | частично |
| B-iii | **Марков на состояниях ног** (переиспользуем паттерн проекта, перенацелив со «зона→зона» на «свинг→свинг») | распределение остаточной структуры | project `ZoneSequenceAnalyzer` (паттерн) + numpy | фаза B | да (механика) |
| B-iv | **Early Classification of TS (ECTS)** — trade-off точность/ранность | «когда можно решать» | **sktime** `classification.early_classification` | фаза данных | ❌ нужен объём |
| B-v | **Survival / hazard** для остаточной длительности | «сколько зона ещё продлится» | lifelines; statsmodels | фаза B | ⚠️ |
| B-vi | Регрессия остаточных целей (harvest/legs) от префикс-признаков | точечный прогноз структуры | sklearn; project `ZoneRegressionAnalyzer` | фаза данных | ❌ нужен объём |
