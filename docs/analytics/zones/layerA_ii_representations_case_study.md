# Слой A-ii — Multi-view классификация зон (сравнение представлений)

Отчёт по `research/notebooks/09_layerA_ii_representations.py --no-trap` (2026-07-06).
Шаг A-ii программы `swing_structure_research_program.md`; метод-стек —
`method_tool_stack.md`.

**Рамка:** фаза метода на малых данных — **только механика, без статвыводов**.
Цель шага: собрать multi-view пайплайн и проверить, **робастна ли структура зон к
выбору представления** (не «найти классы»).

## Что построено

Три представления зоны (лестница абстракции, direction-normalized), общий набор
**46 зон** (1h 21 / m15 25, `narrow_zone`):
- `leg_seq` — последовательность знаковых амплитуд ног (var-length) → **DTW** (tslearn);
- `macd_path` — путь MACD-линии, ресемпл L=24 → **DTW** + **catch22** (pycatch22);
- `price_path` — favorable-нормированный ценовой путь → **DTW** + **catch22**.

Согласие партиций меряется **adjusted Rand index** (ARI); k=3.

## Метод-фикс: z-нормировка перед DTW

Первый прогон: DTW клал **все 25 зон m15 в один кластер** — DTW чувствителен к
масштабу, а амплитуды/пути 1h и m15 разного масштаба → кластеризация ловила
**датасет**, не форму. Фикс — **z-нормировка каждого ряда** (`TimeSeriesScalerMeanVariance`).
После неё оба датасета размазаны по кластерам (артефакт снят). Это переиспользуемый
урок для фазы данных.

## Результат (после снятия артефакта)

ARI между разбиениями (0 = случайно, 1 = идентично):

| | DTW leg | DTW macd | DTW price | c22 macd | c22 price |
|---|---|---|---|---|---|
| **DTW leg_seq** | 1.00 | −0.01 | 0.19 | −0.01 | 0.15 |
| **DTW macd_path** | | 1.00 | −0.02 | **0.34** | −0.01 |
| **DTW price_path** | | | 1.00 | 0.01 | **0.36** |
| **c22 macd_path** | | | | 1.00 | 0.12 |
| **c22 price_path** | | | | | 1.00 |

Среднее попарное ARI = **0.11**.

- **Согласие только на одном и том же ряде, между методами** (DTW↔catch22 на macd_path
  0.34; на price_path 0.36) — внутри представления слабая структура есть.
- **Между разными представлениями** (ноги vs MACD vs цена) ARI ≈ 0 — уровни абстракции
  режут зоны по-разному.

## Вывод (честно)

1. **Устойчивой, инвариантной к представлению классификации зон на n=46 нет.** Ни одно
   представление не даёт воспроизводимо доминирующую структуру; кросс-view согласие
   слабое (0.11).
2. Это ожидаемо на малых данных и **подтверждает дата-гейт**: робастные классы (и тем
   более прогноз, слой B) требуют фазы больших данных.
3. **Механика метод-стека работает** (multi-view, DTW, catch22, ARI, репликация) — цель
   фазы метода достигнута. Найден реальный фикс (z-норма перед DTW).
